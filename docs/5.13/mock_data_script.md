# 2026-05-13 Mock Data Script Memo

## 対象

- `uv run generate-mock-data --config ...`
- 実装本体は `src/scripts/generate_mock_data.py`

## 出力仕様

- 生成ごとに `data/raw/YYYYMMDDHHMMSS/` を作成する
- その配下に設定ファイルで指定した CSV 名を保存する
- 現在の想定ファイル名
  - `mock_acc_only.csv`
  - `mock_with_gyro.csv`
- 既存 CSV を毎回上書きしない運用にする

## CSV カラム

- 共通
  - `timestamp`
  - `elapsed_ms`
  - `acc_x`
  - `acc_y`
  - `acc_z`
  - `label`
- ジャイロ ON のときだけ追加
  - `gyro_x`
  - `gyro_y`
  - `gyro_z`

## ラベル定義

- `0`: 平坦
- `1`: 段差
- `2`: 粗さ

## 時系列生成ルール

- サンプル間隔は初期値 `0.25s` で `4Hz`
- ラベルは `0 -> 1 -> 2` の固定順ではなくランダム遷移
- ただしサンプル単位ランダムではなく，区間単位で継続時間を持たせる
- 初期の継続時間レンジ
  - `0=平坦`: `5s - 20s`
  - `1=段差`: `0.25s - 1.0s`
  - `2=粗さ`: `3s - 10s`
- 初期のラベル重み
  - `0`: `0.55`
  - `1`: `0.15`
  - `2`: `0.30`

## 波形の考え方

- `平坦`: 低振幅で緩やかな揺れ
- `段差`: 短時間の大きいスパイク + 反動
- `粗さ`: 細かい高周波揺れが連続
- ジャイロは完全独立ではなく，加速度変化にある程度相関させる

## 段差の扱い

- いまのモック生成では，`段差` は「段差が始まってから収束するまで」の短い瞬間イベント寄りとして扱う
- そのため `粗さ` のような連続区間よりも短く設定する
- 初期値では `0.25s - 1.0s` とし，必要なら実データ収集後に再調整する

## 現時点でこの定義を採る理由

- 初期フェーズでは，段差の接触瞬間だけでなく，その前後を含めたセンサ応答をまとめて扱う方が学習評価パイプラインに乗せやすい
- `4Hz` 条件では，段差の接触瞬間だけを切り出すとサンプル数が少なすぎて扱いにくい
- そのため，衝撃が出てから少し揺れ返し，通常状態へ戻るまでを 1 つの短いイベントとして定義する
- これは厳密な物理再現ではなく，初期の特徴量抽出と分類実験を成立させるための実務上の定義である

## 今後の改善候補

- 実データを確認して，段差イベントの継続時間を再推定する
- `段差` を「接触瞬間」と「余韻込みイベント」に分けるか検討する
- `平坦` と `粗さ` をベース区間にして，その上に `段差イベント` を挿入する生成方式へ拡張する
- `4Hz` と `50Hz` で同じ秒数定義を使わず，サンプリング周波数ごとに継続時間設定を分ける
