# Roadmap - 2026-05-13

## Goal

- 路面不整検知の学習評価パイプラインを段階的に成立させ、比較可能な実験系を作る
- 最終的にモバイル推論へ接続できるモデル出力まで到達する

## Scope

- 直近はローカル環境でのデータ生成、特徴量抽出、学習、評価を対象にする
- モバイル実装そのものはスコープ外とし、ONNX 出力準備までを対象にする

## Phase 1: ベースライン学習評価パイプラインの成立

目的:

- モックデータでデータ生成から学習評価までを通しで動かす

実施項目:

- モックデータ生成スクリプトの整備
- 4Hz 条件の再現
- ジャイロ ON/OFF 切替の実装
- スライディングウィンドウ特徴量抽出の実装
- SVM / Random Forest の学習スクリプト整備
- Recall / Precision / Confusion Matrix 出力の整備

成果物:

- 比較実験に使えるモックデータ
- 学習済みモデル
- 評価レポートの初版

完了条件:

- パターンAを最低1回通しで実行でき、評価結果を保存できる

## Phase 2: ジャイロ追加のアブレーション

目的:

- ジャイロ追加の純粋な効果を確認する

実施項目:

- 4Hz 条件でパターンAとパターンBを比較
- 誤分類傾向の差分確認
- 寄与特徴量の確認

成果物:

- 加速度のみと加速度+ジャイロの比較表
- 誤分類傾向の整理メモ

完了条件:

- 加速度のみとの比較表が作成できる

## Phase 3: 高サンプリング化の検証

目的:

- 50Hz 化による検知性能の改善余地を検証する

実施項目:

- 50Hz モックデータ生成
- 4Hz 系との差分評価
- 窓幅や特徴量の再調整

成果物:

- 4Hz 系と 50Hz 系の比較結果
- 高サンプリング条件での特徴量設計メモ

完了条件:

- 4Hz 系と 50Hz 系の評価差を説明できる

## Phase 4: モバイル移植準備

目的:

- 学習済みモデルを推論可能な形式でエクスポートする

実施項目:

- 学習済みモデルの ONNX 変換
- 推論入出力仕様の固定
- モバイル側実装への受け渡し情報整理

成果物:

- `.onnx` モデル
- 推論入出力仕様書

完了条件:

- 少なくとも1モデルを `.onnx` として出力できる

## Immediate Next Actions

- `src` 配下の現行スクリプトを Phase 1 の流れに沿って責務分割する
- モックデータ生成条件を設定ファイル化する
- 評価結果の保存先と出力形式を固定する
- 比較実験の実行単位を README か別ドキュメントで明文化する

## Risks

- モックデータで得た精度が実データへそのまま一般化しない可能性がある
- 4Hz と 50Hz の結果を同じ軸で解釈すると誤る可能性がある
- SVM の解釈性は限定的であり、特徴量寄与の説明方法を別途整理する必要がある
