# 2026-07-12 車椅子実測データ 6.22 vs 7.12 比較考察

## 概要

`wheelchair_data/7.12` で新たに収集したデータについて、有効/無効セッションを仕分けたうえで `wheelchair_data/6.22` と同一条件（window 1.0秒 / stride 0.25秒 / ラベル前倒し 0.5秒）でデータセット化し、段差（dansa）・凹凸（dekobo）それぞれの特性を比較した。

## 前提: セッションの有効/無効仕分け

`wheelchair_data/7.12/sessions_index.csv` および各セッションディレクトリの `session_result.csv` の `result` 列に基づき仕分けた。

- 有効（`success`）: 13 件
- 無効（`failed` / `reason=button_miss`）: 2 件
  - `20260711201737`（duration 2.6秒）
  - `20260711201958`（duration 3.8秒）

いずれも操作ミスで即終了しており、実データとして使用不可と判断。

## 前提: `weight_kg` カラムと可視化への反映

`sessions_index.csv` の最終カラムは体重（kg）。`plot_wheelchair_data.py` / `plot_world_vertical_data.py` / `plot_world_z_only.py` に `load_weight_kg()` を追加し、`session_result.csv` から体重を読み取ってプロットのファイル名・タイトルに付記するようにした（`world_csv` のディレクトリ構成は `build-wheelchair-dataset` との互換性維持のため変更していない）。

- 51.0kg: 11 run
- 71.0kg: 2 run（うち 1 run はデータ不備で後述の通り除外）

## データセット構築

- 6.22 側: `data/features/20260630092253/wheelchair_world_z_dataset.csv`（前倒し0.5秒版、既存資産）
- 7.12 側: `configs/wheelchair_world_z_svm_712.yaml` を新規作成し `data/features/20260712215522/wheelchair_world_z_dataset_712.csv` を生成

### データ品質問題: `20260711202058` を除外

`20260711202058`（71kg, dekobo収録セッション）は `dekobo_start` はあるが `dekobo_stop` が存在せず、録音がイベント継続中にそのまま終了していた（`dekobo_start` からの経過時間とセッション終了時刻がほぼ一致）。`build_event_intervals` は start/stop の対応を必須とするため、今回の集計からは除外した（12 run で構築）。

## 0. 収集デザインの違い

| | 6.22 | 7.12 |
|---|---:|---:|
| 有効run数 | 11 | 12（除外1件込みで13） |
| 1runあたり継続時間（平均/中央値） | 91.6秒 / 96.6秒 | 16.7秒 / 11.3秒 |
| 1run内のラベル構成 | flat+dansa+dekoboが混在 | ほぼ単一目的（dansa専用 / dekobo専用 / flatのみ） |

6.22は長時間走行の中で複数イベントに遭遇する形、7.12は短時間で狙った1イベントのみを記録する形に収集方法が変化している。

## 1. 段差（dansa）

| | 6.22 | 7.12 |
|---|---:|---:|
| イベント回数 | 5回 | 4回 |
| 継続時間（平均/中央値） | 2.20秒 / 2.30秒 | 2.04秒 / 2.29秒 |
| window数 | 53 | 40 |
| z_range平均 | 19.45 | 22.40 |
| z_diff_max平均 | 12.79 | 15.95 |
| z_std平均 | 7.48 | 8.87 |

継続時間はほぼ同じ（2秒前後）で物理特性は再現されている。振幅は7.12がやや大きいが、flatベースラインのノイズも上がっているため単純な比較はできない（3節参照）。イベント数は6.22時点から一貫して少なく（4〜5回）、7.12でも改善していない。

## 2. 凹凸（dekobo）

| | 6.22 | 7.12 |
|---|---:|---:|
| イベント回数 | 17回 | 3回 |
| 継続時間（平均/中央値） | 5.35秒 / 4.88秒 | 8.46秒 / 7.97秒 |
| window数 | 368 | 102 |
| z_range平均 | 23.68 | 25.04 |
| z_diff_max平均 | 15.92 | 15.89 |
| z_std平均 | 9.15 | 9.72 |

7.12はdekoboの回数は6.22の1/5程度に減った一方、1回あたりの継続時間は約1.6倍。短い断続的なガタガタ道を何度も踏む6.22に対し、荒れた区間をまとめて長く記録する7.12という収集スタイルの違いが表れている。z_diff_max平均はほぼ同一で、凹凸の物理特性は安定して再現されている。ただしイベント回数が3回のみで、run単位の偏りが全体平均にそのまま出るリスクがある。

## 3. flatベースラインノイズの違い

| | 6.22 | 7.12 |
|---|---:|---:|
| flat z_range平均 | 13.20 | 15.94 |
| flat z_std平均 | 5.10 | 6.16 |

7.12はflat（平坦路）のノイズレベルが6.22より2割ほど高い。flat基準でのコントラスト（イベント/flat比）:

| ラベル/flat比 | 6.22 | 7.12 |
|---|---:|---:|
| dansa z_range比 | 1.47倍 | 1.41倍 |
| dekobo z_range比 | 1.79倍 | 1.57倍 |

7.12の方がイベント/flatのコントラストがやや小さく、識別がわずかに難しくなっている可能性がある。原因はスマホの持ち方・端末固定方法・走行路面そのものの違いなどが考えられるが断定はできない。モデル評価時に注意すべき点。

## 4. 体重に関する注意点

体重による振動特性の比較は現状不可能。71kgの有効runは `20260711201934`（dekoboのみ48 windows）の1本のみで、`20260711202058` は上記の理由で除外済み。51kgとの比較を論じるにはサンプルが著しく不足している。

## まとめ

- 段差＝短時間・強い立ち上がり、凹凸＝長時間・持続的な振動という物理特性自体は6.22/7.12で一貫して観測でき、収集方法が変わっても信号の性質は再現されている。
- 一方で7.12は (1) イベント絶対数がさらに少ない、(2) flatのベースラインノイズが上がりコントラストがやや低下、(3) 71kgデータの欠陥runと体重サンプル不足、という3点が今後の学習に影響しうる。

2. 6.22 + 7.12 を結合した統合データセットでのモデル再学習・評価
