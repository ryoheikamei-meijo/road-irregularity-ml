# ADR - 2026-05-13

## Status

- Draft

## Title

- 路面不整検知MLパイプラインの初期方針

## Context

このプロジェクトでは、スマートフォン等で取得できるセンサーデータを用いて、路面状態を分類する機械学習パイプラインを構築する。現時点で確定している前提は以下の通り。

- まずはローカル環境で、特徴量抽出、学習、評価までを一気通貫で動かす。
- 主対象の入力は加速度3軸であり、拡張としてジャイロ3軸を追加する。
- 先行研究との比較可能性を保つため、比較条件を揃えた評価系を明示的に持つ。
- モバイル組み込みは最終目標に含むが、直近では学習評価パイプラインの成立を優先する。

## Decision

### 1. 開発環境

- パッケージ管理は `uv` を採用する。
- 学習・評価の主要ライブラリは `scikit-learn`、データ加工は `pandas` と `numpy` を使う。
- 深層学習フレームワークは現段階では導入しない。

理由:

- 現フェーズは仮説検証と特徴量設計が主目的であり、SVM と Random Forest を素早く回せる構成が最も合理的なため。

### 2. 学習対象データの扱い

- 初期実装ではモックデータを用いて学習評価パイプラインを成立させる。
- データ形式は先行研究のログ形式をベースにしつつ、ジャイロ3軸を拡張する。
- 想定カラムは以下とする。
  - `timestamp`
  - `elapsed_ms`
  - `acc_x`, `acc_y`, `acc_z`
  - `gyro_x`, `gyro_y`, `gyro_z`
  - `label`

ラベル定義:

- `0`: 平坦
- `1`: 段差
- `2`: 粗さ

### 3. 比較検証の基本原則

- 先行研究との定量比較のため、まずは先行研究と同じ 4Hz 相当、すなわち 250ms 間隔の条件で評価する。
- その後に 50Hz 条件を追加し、高解像度化による性能差を別軸で検証する。
- センサー追加の効果とアルゴリズム変更の効果を分離するため、ジャイロ入力は ON/OFF を切り替え可能にする。

この原則により、以下を独立に評価できる。

- ルールベース閾値判定から ML へ置き換えた効果
- 加速度のみから加速度+ジャイロへ拡張した効果
- サンプリング周波数を上げた効果

### 4. 特徴量抽出方針

- 生データをスライディングウィンドウで区切って特徴量化する。
- 初期の窓幅は 1.0 秒程度を基準とする。
- 特徴量は以下を最低限含む。
  - 移動平均
  - RMS
  - 分散
  - 最大値
  - 最小値

### 5. 初期モデル方針

- 初期比較対象モデルは `SVM` と `Random Forest` とする。
- 評価パターンは少なくとも以下の2系統を持つ。

パターンA:

- 入力は加速度3軸のみ
- 先行研究相当条件との比較基準として扱う

パターンB:

- 入力は加速度3軸 + ジャイロ3軸
- 提案手法として扱う

### 6. 評価方針

- 主要指標は Recall と Precision を用いる。
- 混同行列を作成し、とくに「段差」と「粗さ」の誤分類傾向を確認する。
- Feature Importance 等を用いて、どの軸が分類に寄与したかを確認する。

### 7. モバイル移植方針

- 学習済みモデルは `skl2onnx` で `.onnx` に変換する前提とする。
- モバイル側の推論実行は ONNX Runtime を想定する。
- ただし直近ではモバイル実装そのものは優先せず、ローカルでの学習評価パイプライン完成を優先する。

### 8. 代替案の扱い

- ONNX Runtime が要件に合わない場合に備えて、`m2cgen` 等によるコード変換をバックアップ案として保持する。
- ただし、現時点では主経路として採用しない。

## Consequences

### 良い点

- 比較条件が明確になり、研究的な説明責任を満たしやすい。
- モックデータから始めることで、データ収集前でもパイプライン全体の不整合を早期に潰せる。
- モバイル対応を後段に回すことで、今やるべき実装範囲が絞られる。

### 注意点

- モックデータで精度が出ても、実データへの一般化性能は別途検証が必要。
- 4Hz と 50Hz は同一比較軸ではないため、評価レポートでは目的を分けて扱う必要がある。
- Feature Importance はモデル依存のため、SVM では解釈方法を事前に整理しておく必要がある。

## Open Questions

- 先行研究の固定閾値判定を、このリポジトリ内でどこまで再現するか
- Feature Importance を SVM に対してどの指標で説明するか
- 実データ収集時のラベル付与ルールをどの粒度で設計するか
